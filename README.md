## Why this repo?

Solutions are grouped by **pattern** (not by solve date), so you can study one technique at a time. Each file is written for **humans**: problem restatement, intuition, and complexity — not just code a judge accepts. The **tracker and README tables refresh from the same source** when you run `python update_tracker.py` locally or when GitHub Actions runs on push.

## How to navigate

1. **Pick a pattern folder** that matches the topic you are weak on (e.g. `sliding-window/`), then read solutions in the order that matches your roadmap.
2. **Open any `.java` file**: the block comment at the top is the study guide; the code is the minimal LeetCode-style class.
3. **Run `python update_tracker.py`** after adding files so `tracker.json`, the progress table, and pattern counts stay accurate (CI does this on every push to `main` or `master`).
